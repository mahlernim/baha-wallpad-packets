Samsung BAHA / Honeywell MC200-50

Heater Packet Notes

This document is a human-readable summary of the RS485 heater packets currently mapped on the BAHA home network. It records the confirmed heater frame meanings, room mapping, and working control writes.

Node 40 90 UART 9600 8N1 Checksum XOR of payload bytes Known Actions Away / Clear Away / Set Temperature

What Is Proven

  • The heater-related BAHA node is 40 90.
  • 01 -> 81 is a poll/response pair for the 5-room current-temperature table.
  • 05 -> 85 is a poll/response pair for the 5-room target-setpoint table.
  • 02 06 00 ... is a real action family.
  • A one-hot C0 in a room slot sets that room to away / frost-protect mode.
  • A one-hot 80 in a room slot clears away on all five confirmed room slots.
  • A normal-coded room byte such as 95 sets that room's normal target temperature when T0=00, and from away it also exits away in one step.
  • T0=02 is a restore-to-stored-setpoint control for away rooms and has no observed effect once the room is already normal.
  • 02 40 90 80 00 52 00 behaves like a generic receipt / ACK for the action family.

Final Conclusions

  • The 81 table is the current measured-temperature table.
  • The 85 table is the target setpoint table.
  • Reply subtype 02 means not-all-away, and subtype 03 means all rooms away.
  • The heater write path is fully exposed on 40 90 with room-specific 02 06 ... action frames.

Frame Shape

All observed heater packets use the same framing style:

02 [node_hi] [node_lo] [opcode] [payload...] [xor_checksum] 00
Byte Meaning Observed Heater Values
00 Start byte 02
01 02 Node / function 40 90
03 Opcode 01, 05, 02, 81, 85, 80
Last-1 XOR checksum Computed over all bytes from the first 02 through the last payload byte
Last End byte 00
Example checksum: for 02 40 90 05 00 D7 00, the checksum is 02 ^ 40 ^ 90 ^ 05 ^ 00 = D7.

Reply Field Meaning

A BAHA heater reference provided later matches the opcode families exactly and labels the payload as DL plus a sequence of Tn bytes used for mode and temperature. For this controller, the captures still show DL=06, which means the practical payload is T0..T5, not T0..T7.

04 Length / subtype marker Always observed as 06 on heater replies

What The Heater Means By "Off"

On this system, the user-visible heater "off" behavior does not currently look like a separate hard-off binary state. It behaves much more like away / frost-protect at 10 C.

Practical rule: if the wallpad says "off", the BAHA-side packets currently look like "away 10 C", not "output completely disabled with no temperature target".

Earlier Failed Write Guesses

Several direct write guesses were tested and failed to change the 85 setpoint table.

  • 06 often triggered an immediate 85 readback but no state change.
  • 04 and 07 did not produce useful state changes.
  • That makes a simple direct bulk write to 40 90 unlikely.

Confirmed Temperature-Set Write

Confirmed working format: 
02 40 90 02 06 00 [room1] [room2] [room3] [room4] [room5] [chk] 00
where a non-zero room byte sets that room's normal setpoint and a zero byte leaves that room unchanged.

The room byte uses the same normal temperature encoding already seen in the 85 setpoint table: 

encoded_setpoint = 0x80 + temperature_celsius

Confirmed master-bedroom examples

20 C = 02 40 90 02 06 00 00 94 00 00 00 42 00
21 C = 02 40 90 02 06 00 00 95 00 00 00 43 00

Observed proof

before : 02 40 90 85 06 02 93 94 94 94 88 DC 00
write  : 02 40 90 02 06 00 00 95 00 00 00 43 00
ack    : 02 40 90 80 00 52 00
after  : 02 40 90 85 06 02 93 95 94 94 88 DD 00

That changed the master bedroom from 20 C to 21 C. Sending 02 40 90 02 06 00 00 94 00 00 00 42 00 then changed it back to 20 C.

What T0 appears to mean now

Value Current Interpretation Status
T0=00 Working action/write path for away, clear-away, and normal setpoint changes Confirmed
T0=02 Not a direct normal setpoint write. On tested rooms it restored an away room to its stored normal setpoint and ignored the supplied temperature; once already normal it had no observed effect. Confirmed in live tests
T0=03 All-away reply subtype in read responses Confirmed in replies

Best sequence if the room is currently away

restore stored normal setpoint: 02 40 90 02 06 00 00 80 00 00 00 56 00
set a new 20 C target directly: 02 40 90 02 06 00 00 94 00 00 00 42 00
The live tests closed the heater-side protocol questions: room order, current table, away subtype, clear-away behavior, and direct setpoint writes are all confirmed.

Confirmed Setpoint Limits

The master bedroom was used to test the numeric limits of the direct setpoint write path. The accepted range is now confirmed as 5 C through 35 C.

Test Write Observed Result
5 C 02 40 90 02 06 00 00 85 00 00 00 53 00 Accepted. Readback changed to master 85, meaning 5 C.
4 C 02 40 90 02 06 00 00 84 00 00 00 52 00 Rejected logically. ACK returned, but the setpoint stayed at 5 C.
35 C 02 40 90 02 06 00 00 A3 00 00 00 75 00 Accepted. Readback changed to master A3, meaning 35 C.
36 C 02 40 90 02 06 00 00 A4 00 00 00 72 00 Rejected logically. ACK returned, but the setpoint stayed at 35 C.
Practical limit rule: the controller will acknowledge out-of-range writes with the usual 02 40 90 80 00 52 00, but the 85 setpoint table will not move past 5 C or 35 C.
